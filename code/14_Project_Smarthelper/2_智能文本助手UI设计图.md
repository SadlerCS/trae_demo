# 智能文本助手 UI 文本设计图

## 设计目标
- 页面简洁、操作直观，核心动作优先：翻译与朗读。
- 悬浮不遮挡内容，结果就地可见，尽量减少鼠标移动与点击。
- 统一、友好的中文文案，状态清晰、可恢复。

## 信息架构
- 悬浮操作条（选中文本后出现）
- 翻译结果浮层（在操作条下方）
- 朗读控制浮层（最小化控件）
- 浏览器弹出页（Popup）
- 设置页（Options）
- 首次引导与权限提示
- 通知与错误提示

## 悬浮操作条
- 标题（隐藏）：智能文本助手
- 按钮：
  - 翻译
  - 朗读
- 目标语言切换：中文 | 英文
- 工具提示：
  - 翻译选中的文本
  - 朗读选中的文本
  - 切换目标语言

## 翻译结果浮层
- 标题：翻译结果
- 状态文案：
  - 正在翻译…
  - 翻译失败，请重试
  - 未检测到选中文本
  - 未配置 Kimi API，前往设置
- 结果区域：显示译文
- 操作：
  - 复制
  - 朗读
  - 关闭
- 工具提示：
  - 复制翻译结果
  - 关闭面板

## 朗读控制浮层
- 控件（最小化）：
  - 播放
  - 暂停
  - 停止
- 扩展设置（可折叠）：
  - 语速
  - 音量
  - 声音
- 状态文案：
  - 正在朗读…
  - 已暂停
  - 已停止朗读
  - 浏览器朗读不可用

## 浏览器弹出页（Popup）
- 标题：智能文本助手
- 开关：选中文本时显示悬浮按钮（开/关）
- 默认目标语言：中文 | 英文
- Kimi API 状态：已配置 / 未配置
- 操作按钮：
  - 打开设置
  - 帮助
  - 反馈
- 说明：选中页面上的文字即可使用翻译或朗读

## 设置页（Options）
- 标题：设置
- 分组：Kimi API
  - 输入框占位：粘贴你的 Kimi API Key
  - 按钮：测试并保存
  - 状态：测试成功 / 测试失败，请检查 Key
- 分组：默认行为
  - 默认目标语言：中文 | 英文
  - 悬浮按钮样式：紧凑 / 标准
- 分组：朗读偏好
  - 默认声音
  - 默认语速
  - 默认音量
- 底部操作：保存
- 保存反馈：
  - 已保存
  - 保存失败，请重试

## 首次引导与权限
- 首次提示：选中任意文本，试试“翻译”或“朗读”
- 权限说明：需要访问当前页面文本以提供翻译与朗读服务
- 权限操作：
  - 同意并启用
  - 以后再说
- 权限结果：
  - 已启用智能文本助手
  - 您可在设置中重新开启权限

## 通知与错误提示
- 已复制到剪贴板
- 网络异常，请检查网络后重试
- 请求超时，请稍后重试
- 服务暂不可用，请稍后重试

## 工具提示（统一）
- 翻译选中的文本
- 朗读选中的文本
- 切换目标语言
- 复制翻译结果
- 关闭面板

## 文案规范
- 语言风格：
  - 简洁直白，动作导向；避免冗长与技术术语。
  - 状态可感知、可恢复，失败必给出行动指引。
- 命名与大小写：统一中文；英文仅出现在目标语言选择与技术名词（Kimi API）。
- 可访问性：
  - 按钮文本可朗读；状态提示与动作可被屏幕阅读器识别。
  - 重要提示使用简洁句式与明确动词（如“复制”“关闭”）。
- 国际化：
  - 目标语言仅中文/英文；其他语言后续扩展。
  - 所有文案可抽取为字典，支持后续多语言。

## 交互示例（摘要）
- 选中文本 → 悬浮操作条出现：翻译 | 朗读 | 中文/英文
- 点击“翻译” → 显示“正在翻译…” → 成功显示“翻译结果”与复制/朗读/关闭
- 点击“朗读” → 显示朗读控件（播放/暂停/停止）与“正在朗读…”
- 未配置 API → 在结果浮层显示“未配置 Kimi API，前往设置”并提供按钮入口
